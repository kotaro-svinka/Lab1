## Описание проекта
Данный проект представляет собой реализацию иерархии классов для электронных гаджетов. Работа выполнена в рамках изучения принципов объектно-ориентированного программирования (ООП).

## Иерархия классов
Проект включает в себя три уровня классов:
1. **Gadgets (Абстрактный класс)** — базовый уровень, содержащий общие характеристики (бренд, модель, год выпуска).
2. **ElectronicDevice (Абстрактный класс)** — промежуточный уровень, добавляющий специфические поля для электроники (вес, емкость батареи, наличие беспроводной связи).
3. **Конечные классы:**
   - `Watch` (Часы)
   - `Smartphone` (Смартфон)
   - `Laptop` (Ноутбук)

## Реализованные критерии (ООП)
В работе продемонстрированы основные принципы ООП:

*   **Абстракция:** Использование абстрактных классов и методов (`turnOn`, `turnOff`, `charge`), которые реализуются в дочерних классах.
*   **Инкапсуляция:** Все поля классов объявлены как `private`. Доступ к данным реализован через публичные **геттеры и сеттеры**.
*   **Наследование:** Создана двухуровневая цепочка наследования с использованием ключевых слов `extends` и `super`.
*   **Полиморфизм:**
    - **Переопределение (Overriding):** Каждый класс по-своему реализует методы включения и зарядки.
    - **Перегрузка (Overloading):** Реализованы методы с одинаковыми именами, но разными параметрами (например, установка будильника или запуск программы с разными аргументами).

## Особенности реализации
- **Конструкторы:** Для каждого класса созданы конструкторы с параметрами и конструкторы по умолчанию.
- **Статические поля:** В классе `Watch` реализован статический счетчик (`watchCount`), который отслеживает количество созданных объектов данного класса.
- **Вывод данных:** Класс `Main` содержит демонстрацию работы всей иерархии, создавая объекты и вызывая их методы.
